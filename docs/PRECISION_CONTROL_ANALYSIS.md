# 정밀 제어 수준 분석 — 회전축 마모 감소 · 5축 · 자율선박/쇄빙선

자율선박/쇄빙선 시스템이 **정밀 제어 시스템**으로서 어느 수준인지, **회전축 제어·마모 감소**가 왜 중요한지, **5축 정밀 제어**가 엔지니어링 가능한지 정리한다.

---

## 1. 회전축 제어·마모 감소가 중요한 이유

### 1.1 선박/쇄빙선에서의 위치

- **추진축(propulsion shaft)**은 엔진 출력을 프로펠러로 전달하는 단일 회전축이다.  
  파도·얼음 충돌·조류에 의해 **토크 스파이크·위상 흔들림**이 반복되면 베어링·축 이음부 마모와 피로가 누적된다.
- **마모 감소**는 다음을 직접 줄인다:
  - 드라이독 입거 주기 (비용·다운타임)
  - 예기치 않은 고장·안전 리스크
  - 쇄빙 시 램(ramming) 반복으로 인한 축·베어링 수명 단축

따라서 **“회전축 제어로 마모율을 낮추는 것”**은 선박·쇄빙선 모두에서 **유지보수·신뢰성·비용** 측면에서 중요하다.

### 1.2 현재 엔진이 하는 일

- **OrbitStabilizer**: 회전 **위상(각도) φ**의 궤적을 기억하고, 미래 위상 예측으로 **선제적 보정력**을 계산 → 기존 PID 앞단에 보정항만 추가.
- **Marine Propulsion**: 위 로직을 선박 추진축에 적용 → **마모 31.9% 감소** (10,000 사이클 검증), Fail-Safe 0.1ms.

즉, “정밀도”는 **위상(각도) 제어 정밀도**와 **보정 타이밍(응답/Fail-Safe)**으로 정의된다.

---

## 2. 현재 사양이 “정밀 제어” 수준인지

### 2.1 문서에 명시된 수치

| 항목 | 값 | 출처 |
|------|-----|------|
| **위상(각도) 정밀도** | **0.1도 이하** | Marine Propulsion 기술 백서 §6 |
| **응답 시간** | **< 100ms** | 선박용 요구 충족 |
| **제어권 이양(Fail-Safe)** | **0.1ms 이내** | 기술 백서 §6, OrbitStabilizer README |
| **궤도 안정도** | 0.850 (0~1) | 외란 후 복구 < 100ms |

### 2.2 해석

- **0.1도**: 각도 제어 기준으로는 **정밀 제어**에 해당한다.  
  (일반 선박/조선은 0.5~1도급, CNC/로봇은 0.01~0.1도급 — 현재는 그 중간~상단.)
- **100ms**: 선박·쇄빙선의 **추진/조타 루프** 주기에 비해 충분히 빠른 편이다.  
  (보통 100~500ms 제어 주기 사용.)
- **0.1ms Fail-Safe**: 이상 시 **즉시** 기존 PID로 넘기는 수준이라, 안전·신뢰성 측면에서 **정밀·산업 제어**에 요구되는 수준을 만족한다.

**결론**:  
**추진축 1축(회전 위상)**에 대해서는 **정밀 제어 시스템**으로 볼 수 있다.  
다만 제어 대상이 “전체 선박 6-DOF”가 아니라 **“1개 회전축 위상”**이므로, “정밀 제어”는 **그 1축**에 한정된다.

## 3. 5축 정밀 제어가 가능한지

### 3.1 “5축”이 의미하는 것

- **일반 공작기계/로봇**:  
  - 5축 = 위치 3축 (X,Y,Z) + 회전 2축 (A,B)  
  - 또는 선박 맥락: **선박 3-DOF (ξ, η, ψ) + 추진 1축(φ) + 보조 1축(예: 트림/리스트)** 등으로 해석 가능.
- **동일 엔진 생태계 내**:  
  **Grid Engine**이 이미 **5D(위치 3 + 회전 2)**를 “정밀 운동 제어 엔진”으로 정의하고, 5축 CNC·일반 로봇 움직임에 대응하도록 설계되어 있다.

### 3.2 자율선박/쇄빙선에의 매핑

| 축 | 물리 의미 | 현재 구현 | 정밀도·비고 |
|----|-----------|-----------|-------------|
| 1 | 위치 ξ (동서) | Potential Field → 경로 | 미터급 (일반 항법) |
| 2 | 위치 η (남북) | Potential Field → 경로 | 미터급 |
| 3 | 방위각 ψ (선수 방향) | 경로 제어 + (확장 시) OrbitStabilizer | 도(°) 단위 가능 |
| 4 | 추진축 위상 φ | **OrbitStabilizer + Marine Propulsion** | **0.1도 이하** (정밀) |
| 5 | (선택) 트림/리스트 또는 보조 회전 | 개념상 Grid 5D 또는 OrbitStabilizer 2번째 채널 | 확장 시 동일 계약 적용 가능 |

- **추진 1축(φ)**: 이미 **정밀 제어(0.1도, 0.1ms Fail-Safe)** 수준.
- **나머지 4축(ξ, η, ψ + 1)**:  
  - **경로(ξ, η, ψ)**는 보통 “정밀 제어”보다는 “항법·경로 추종”에 가깝고,  
  - **5축 전체를 정밀화**하려면 **Grid 5D**를 선박 상태 (ξ, η, ψ, φ, 5번째 축)에 매핑하거나,  
  - **OrbitStabilizer를 축별로 여러 개** 두는 방식(축당 1 DOF 위상)으로 확장하면 된다.

### 3.3 엔지니어링 가능 여부

- **개념·수식**:  
  - Grid Engine 5D = 위치 3 + 회전 2, 단위 계약(rad/deg), 위상 일관성 유지.  
  - OrbitStabilizer = 1 DOF 위상 기억 + 예측 보정.  
  → **5축 = OrbitStabilizer 5개 (축별) 또는 Grid 5D 1개**로 모델링 가능.
- **구현 상태**:  
  - **검증·배포된 것**: 추진축 **1축** (OrbitStabilizer + Marine Propulsion).  
  - **5축**: Grid 5D는 “로보틱스·5축 CNC”용으로 문서·코드 존재, **선박 도메인 매핑**은 자율선박 엔진에서 계약만 정의된 상태.  
  → **정밀제어 5축 제어는 “엔지니어링 가능한 수준”**이며, 동일 생태계(Grid 5D + OrbitStabilizer 확장)로 확장 가능하다.

## 4. 자율선박/쇄빙선 시스템 전체가 “정밀 제어 시스템”인지

### 4.1 하이브리드 구조

- **정밀 제어에 해당하는 부분**  
  - **추진축 위상(φ)**  
    - OrbitStabilizer + Marine Propulsion.  
    - 0.1도, 100ms, 0.1ms Fail-Safe, 마모 감소.  
  - **(확장 시)**  
    - ψ를 “위상”으로 취급해 OrbitStabilizer 1채널 더 적용.  
    - Grid 5D를 (ξ, η, ψ, φ, 5번째)에 매핑한 다축 정밀 궤적 제어.

- **일반 제어에 해당하는 부분**  
  - **경로 계획·항법**  
    - Potential Field, waypoint, 장애물/빙 회피.  
    - 단위: 위치 m, 방위 °.  
  - **속도·추진력 지령**  
    - U_ref, τ_ref — “정밀 위치/각도”보다는 “목표 속도·토크” 수준.

### 4.2 한 줄 정리

- **시스템 전체**를 하나의 “정밀 제어 시스템”으로만 보기는 어렵다.  
- **추진축(및 확장 시 다른 회전축)**: **정밀 제어** (0.1도, 0.1ms Fail-Safe, 마모 감소).  
- **경로·항법**: **일반 제어** (미터·도 단위).  
- 따라서 **“정밀 제어가 필요한 부분(축 제어)”과 “일반 제어 부분(경로)”이 결합된 하이브리드**로 보는 것이 맞다.  
- 쇄빙선의 경우, **램 시 축 충격**을 OrbitStabilizer가 흡수하는 부분이 곧 **정밀(축) 제어**가 중요한 구간이다.
